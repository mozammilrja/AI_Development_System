# docs/

## Purpose

System documentation and technical specifications.

## Contents

- **architecture.md**: System architecture overview
- **developer_guide.md**: Developer onboarding and guidelines
- **user_manual.md**: End-user documentation
- **workflow.md**: Workflow documentation
- **api/**: API documentation (to be generated)
- **adr/**: Architecture Decision Records

## Structure

```
docs/
├── architecture.md     # System design
├── developer_guide.md  # Dev onboarding
├── user_manual.md      # User docs
├── workflow.md         # Workflow docs
├── api/                # API reference
└── adr/                # Decision records
```

## Documentation Standards

- Use Markdown for all documentation
- Keep docs up-to-date with code changes
- Include diagrams where helpful
- Document architectural decisions in ADRs
